# React技術解説ドキュメント

本書は、作成されたソースコード（app.js, KnowledgeDetail.js, KnowledgeList.js）で使用されているReactの機能やWeb技術の仕組みについての解説をまとめたものです。

## 1. app.js に関する疑問
-----------------------------------------

### Q1. 「const { useState, useEffect } = React;」とは何を示しているか

これはJavaScriptの **「分割代入（Destructuring Assignment）」** という構文です。
Reactライブラリのグローバルオブジェクト（またはモジュール）である `React` から、`useState` と `useEffect` という2つのプロパティ（関数）を取り出し、それぞれ同名の定数として定義しています。

* **分割代入をしない場合：**
  コード中で `React.useState` や `React.useEffect` と毎回記述する必要があります。
* **分割代入をする場合：**
  あらかじめ取り出しておくことで、直接 `useState(...)` や `useEffect(...)` と短く書くことができるようになります。

### Q2. useStateを伴う定義では、setKnowledgesなど、配列における右側の変数で引数に値を設定することで、同配列の左側の変数も値が更新されることになるか

**はい、その通りです。** `const [knowledges, setKnowledges] = useState([]);` を例に説明します。

1. **左側の変数（`knowledges`）**：現在の状態（ステート）の「値」を保持します。
2. **右側の関数（`setKnowledges`）**：状態を更新するための「セッター関数」です。

右側の `setKnowledges(新しい値)` を呼び出すと、React内部で管理されている状態が新しい値に更新され、それと連動して左側の変数 `knowledges` に入る値も最新のものに置き換わります。
また、Reactは状態が更新されたことを検知すると、該当するコンポーネントを自動的に **再レンダリング（再描画）** し、画面の表示を最新の状態に更新します。

### Q3. useEffectにはどういう役割があるか

`useEffect` は、Reactコンポーネントにおける **「副作用（Side Effects）」** を実行するためのフック（Hook）です。
副作用とは、コンポーネントの描画（画面表示）処理そのものとは直接関係のない、以下のような外部処理を指します。

* APIやサーバーからのデータ取得（データのフェッチ）
* タイマーの設定（`setInterval`など）や解除
* 手動でのDOM操作
* 外部ライブラリとの連携や購読の設定

`useEffect` は第2引数に「依存配列（Dependency Array）」を指定します。

* `useEffect(コールバック関数, [依存する変数]);`
  依存配列に指定した変数が変化したタイミングで、第1引数のコールバック関数が再実行されます。
* `useEffect(コールバック関数, []);` （空の配列を指定した場合）
  **「コンポーネントが最初に画面に表示された時（マウント時）に1度だけ実行する」** という役割になります。本アプリでは、ページを開いた瞬間にサーバー（`server.php`）から知識一覧データを初期取得するために使用されています。

### Q4. 「fetch('server.php?action=list')」はどういう仕組みの非同期通信か

`fetch` は、ブラウザに標準搭載されている、HTTPリクエストを送信してデータを取得または送信するためのAPIです。

* **非同期通信とは：**
  サーバーと通信してレスポンスを待っている間も、ブラウザの他の処理（スクロールやボタンクリックなどの画面操作）を一時停止（ブロック）させず、バックグラウンドで並行して通信処理を行う仕組みです。
* **処理の流れ：**
  `fetch('server.php?action=list')` は、同じサーバー上にある `server.php` に対して、`action=list` というクエリパラメータを付与した HTTP GET リクエストを送信します。
  `fetch` 関数は即座に **Promise**（非同期処理の完了状態を表すオブジェクト）を返します。通信が成功すると Promise が「解決（resolve）」され、`.then()` や `async/await` 構文を通じてレスポンスデータを受け取ることができます。

### Q5. 「fetch('server.php?action=list')」の関数で記述されているres.json()とdataは同義か

**同義（同じもの）ではありません。それぞれ異なる役割を持っています。**

* `res`（または `response`）：
  サーバーから返ってきた「生のHTTPレスポンス全体」を表すオブジェクトです。ステータスコード（`200 OK` など）やヘッダー情報、およびバイナリやテキストなどの未加工のデータ（ストリーム形式）が含まれています。
* `res.json()`：
  `res` オブジェクトが持つメソッドです。生のレスポンスボディ（中身）からデータを読み込み、JSON形式の文字列をJavaScriptのオブジェクトや配列に変換（パース）します。これも非同期処理であるため Promise を返します。
* `data`：
  `res.json()` による解析（パース）が完了した結果として得られる、 **実際にプログラムで直接操作できる状態になったJavaScriptのデータそのもの**（本アプリの場合は、知識一覧の配列）を指します。

**関係のイメージ：** [生のレスポンス: res] ──(res.json() で解析)──> [解析済みのデータ: data]

### Q6. 「fetch(`server.php?action=detail&file=${encodeURIComponent(knowledge.file_title)}`)」はどういう処理を行う非同期通信か

これは、ユーザーが選択した特定の知識（ナレッジ）の「詳細データ（表示列の設定やテーブルのデータなど）」を取得するために、サーバーへリクエストを送信する非同期通信です。

* JavaScriptのテンプレートリテラル（` ` バックティックと `${}`）を使い、変数 `knowledge.file_title` の値をURLのパラメータに動的に埋め込んでいます。
* サーバーに対して `action=detail` というアクション指定と、`file=ファイル名` というパラメータを送信しています。

#### encodeURIComponentにはどういう役割があるか

`encodeURIComponent` は、文字列をURLのパラメータ（クエリ文字列）として安全に送信できるように **「URLエンコード（パーセントエンコーディング）」** を行うJavaScriptの組み込み関数です。

* **なぜ必要か：**
  URLには使える文字（英数字や一部の記号）が決まっています。日本語の漢字・ひらがな、スペース、あるいはURLの構成要素として特別な意味を持つ記号（`?`、`&`、`=`、`/` など）をそのままURLに含めると、ブラウザやサーバーがURLを正しく解釈できず、文字化けや通信エラーを引き起こします。
* **具体的な動作：**
  例えば「プログラミング 基礎.csv」というファイル名を `encodeURIComponent` に通すと、`%E3%83%97%E3%83%AD%E3%82%B0...` のように安全なアルファベットと記号の組み合わせに変換され、通信が正しく行われるようになります。

### Q7. 「ReactDOM.createRoot(document.getElementById('root'));」にはどういう役割があるか

React 18以降で導入された、Reactアプリケーションをブラウザ上の実際のHTMLに描画（マウント）するための開始地点（ルート）を作成する処理です。

1. `document.getElementById('root')` を使い、HTML側（`index.php` など）に用意されている `<div id="root"></div>` という空の要素（コンテナ）を取得します。
2. `ReactDOM.createRoot(...)` によって、その要素をReactの管理下にある「描画の起点（ルート）」として登録します。
3. このルートに対して `.render(<App />)` を実行することで、Reactコンポーネントのツリーが構築され、ブラウザ上に実際のUIとしてレンダリングされます。

## 2. KnowledgeDetail.js に関する疑問
-----------------------------------------

### Q1. 「const { view_title, view_cols, all_cols, data } = detail;」とは何を示しているか

これもJavaScriptの **「分割代入」** 構文です。
サーバーから取得した詳細データオブジェクト `detail` の中から、以下の4つのプロパティを取り出し、それぞれ同名の定数として一括で定義しています。

* `view_title`：画面に表示するタイトル
* `view_cols`：表示対象として選択されている列のリスト
* `all_cols`：CSVファイルに存在するすべての列のリスト
* `data`：テーブルの各行のデータ（オブジェクトの配列）

これを行うことで、コード内で毎回 `detail.view_title` や `detail.data` のように `detail.` を頭に付けて書く手間を省き、直接 `view_title` や `data` と簡潔に記述できるようになります。

### Q2. 「[...Array(totalPages)].map((_, i) =>」はどういう仕組みのMap関数か

これは、 **「指定した数（`totalPages`）の分だけループを回し、React要素（ページネーションのボタンなど）の配列を作る」** ためのJavaScriptの常套手段（テクニック）です。
仕組みは以下のように分解できます。

1. **`Array(totalPages)`**：
  指定した長さ（例：`totalPages` が 5 であれば 5）の空の配列を作成します。ただし、この時点では「値が設定されていない空枠の並び」です。
2. **`[...Array(totalPages)]`**：
  スプレッド演算子（`...`）を使って空の配列を展開し、新しい配列リテラル `[]` の中に流し込みます。これにより、すべての要素が `undefined` で満たされた「値が実在する配列」に変換されます。（※JavaScriptの仕様上、1の段階の空枠のままだと `map` 関数が要素をスキップしてしまい、ループが実行されません）
3. **`.map((_, i) => ...)`**：
  配列の各要素に対して繰り返し処理を行います。
  * **第1引数の `_`（アンダースコア）**：
    配列の要素の値自体（ここでは `undefined`）を受け取りますが、使用しないため、プログラミングの慣習として「使わない変数」を意味する `_` と書いて無視しています。
  * **第2引数の `i`**：
    配列の現在のインデックス番号（0から始まる連番：`0, 1, 2, 3...`）が自動的に渡されます。
  * **用途**：
    このインデックス `i` を使って、画面上には `i + 1` （1ページ目、2ページ目...）というテキストを持つボタン要素を動的に作り出しています。

## 3. KnowledgeList.js に関する疑問
-----------------------------------------

### Q1. 最初に関数を定義する際に、引数を「{ list, onSelect }」と指定しているのはなぜか

これは、親コンポーネント（`App`）から子コンポーネント（`KnowledgeList`）に渡される **「Props（プロパティ）」** を、分割代入を使って引数部分で直接受け取るための記述です。
Reactでは、親コンポーネントから子コンポーネントにデータや関数を渡す際、それらはすべて一つの `props` というオブジェクトにまとめられて子コンポーネントに渡されます。

* **通常の書き方：**
```javascript
  function KnowledgeList(props) {
    const list = props.list;
    const onSelect = props.onSelect;
    // ...
  }
```

* **分割代入（引数で直接受け取る）書き方：**
```javascript
  function KnowledgeList({ list, onSelect }) {
    // 関数の内部で直接 list や onSelect を変数として使用できます
  }
```

**親コンポーネント側で以下のように子コンポーネントを呼び出している場合：**
  `<KnowledgeList list={knowledges} onSelect={handleSelect} />`
  子コンポーネント側で引数を `{ list, onSelect }` と指定することで、コードが非常にシンプルになり、このコンポーネントが親からどのPropsを必要としているのか（インターフェース）が一目で理解しやすくなるという大きなメリットがあります。
